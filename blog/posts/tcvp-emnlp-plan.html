TCVP 리뷰 분석과 개선 계획

ACL ARR 리뷰어들이 지적한 것들, 리버탈에서 어떻게 대응했는지, 그리고 EMNLP 2026을 타겟으로 무엇을 보강할지 정리한다.

Multimodal · Mar 12, 2026 · 15 min read

리뷰 + 리버탈 한눈에 보기

아이디어 자체(유튜브 타임스탬프 댓글 활용)는 세 리뷰어 모두 인정. 문제는 그 아이디어를 충분히 증명했냐는 것이다.

리뷰어 지적 리버탈 대응 결과
3Z5s, fm3t Novelty 부족 실제 user intent 기반 최초 데이터셋. 쿼리 선호도 70% (Fig. 5) — 말은 했지만 근본적으로 해소 안 됨 ✗ 실질 미해소
3Z5s, BKiQ 데이터 규모 작음 (300 videos) proof-of-concept, 19개 카테고리, 파이프라인 확장 가능 △
3Z5s 9초 윈도우 근거 없음 100샘플 검증: [−9s,+9s]가 97% coverage ✓
3Z5s 쿼리 생성 타당성 LongVALE·Watch&Listen과 동일 방식. 14.3% 노이즈 필터링 ✓
fm3t Training 실험 없음 A100 1장으론 불가, H100 8장+ 필요 — compute 한계 인정 ✗ 미해소
BKiQ 단일 모달리티 캡션 Mixed category 제안 (balance ratio R + floor M). Ablation 포함 △ 리버탈만, 미구현
BKiQ 중복 댓글 처리 타임스탬프별 YouTube 좋아요 수 1위 댓글만 유지 ✓
BKiQ MLLM hallucination 댓글 grounding으로 generic 설명 → 구체적 intent. 예시 제시 ✓
메타 리뷰어 거절 이유 — 리버탈에서 아예 언급 안 함:
Section 4.5: "검색될 가능성이 높은 구간"을 평가한다고 주장 → Appendix B.1 실제 지시문: "more representative and more informative"를 고르라고 함.
측정한 것(콘텐츠 품질) ≠ 주장한 것(search intent). 이 불일치가 결정적 거절 이유.

개선 계획

# 방향 내용 해소하는 지적
1 Visual Query 추가 이미지·클립 형태 입력으로 새로운 태스크 정의. "새로운 태스크의 첫 데이터셋"으로 포지셔닝 → 규모 압박 완화. 방법론은 구체화 필요 Novelty 부족 (핵심)
2 Mixed Modality 구현 리버탈에서 제안만 했던 것을 실제 구현. Appendix A.3 → Section 3으로 편입. Mixed query 평가 추가 단일 모달리티 캡션
3 LanguageBind Fine-tuning zero-shot 평가에서 벗어나 실제 학습 효과 증명. QVHighlights only / +TCVP / TCVP only 세 조합 비교 Training 실험 없음
4 Human Eval 재설계 지시문 수정: "more informative" → "Which moment would you actually search for?". 샘플 40 → 100+. Cohen's κ 보고 메타 리뷰어 핵심 지적
5 데이터 규모 확장 파이프라인 완성, 추가 수집만 하면 됨. 목표: 300 → 1,000+ videos 규모 작음
