TCVP 리뷰 분석과 개선 계획

ACL ARR 리뷰어들이 지적한 것들, 리버탈에서 어떻게 대응했는지, 그리고 EMNLP 2026을 타겟으로 무엇을 보강할지 정리한다.

Multimodal · Mar 12, 2026 · 15 min read

리뷰어들이 지적한 것

리뷰어들은 유튜브 타임스탬프 댓글을 활용하는 아이디어 자체는 참신하다고 인정했다. 하지만 크게 네 가지를 문제로 지적했다.

  • Novelty와 contribution이 부족하다 — 3Z5s, fm3t 공통 지적
  • 데이터셋 규모가 너무 작다 — 3Z5s, BKiQ 공통 지적
  • 캡션 생성이 단일 모달리티에 머문다 — 댓글을 Visual/Audio 둘 중 하나로만 분류해, 두 모달리티가 동시에 관여하는 경우를 처리하지 못함 (BKiQ)
  • Human evaluation이 핵심 클레임과 불일치 — 메타 리뷰어 핵심 지적
메타 리뷰어 거절 이유:
논문의 핵심 클레임은 "TCVP가 실제 사용자의 search intent를 반영한다"는 것인데, Section 4.5에서는 "검색될 가능성이 높은 구간"을 평가한다고 했으면서, 실제 Appendix B.1의 annotator 지시문은 "more representative and more informative"를 고르라고 되어 있다. 측정한 것(콘텐츠 품질)과 주장한 것(search intent)이 다르다.

리버탈에서 대응한 것들

리버탈에서 각 리뷰어의 지적에 대해 아래와 같이 대응했다. 일부는 충분히 해소됐지만, 메타 리뷰어가 지적한 핵심 문제는 리버탈로 커버되지 않았다.

Reviewer 3Z5s

지적대응결과
Novelty 부족 TCVP는 실제 사용자 intent 기반 최초 데이터셋. Figure 5에서 70% 쿼리 선호도 ✓ 대응
데이터 규모 작음 300 videos는 proof-of-concept. 파이프라인 자체가 확장 가능하며 19개 카테고리 커버 △ 부분 대응
9초 윈도우 근거 100샘플 empirical validation: [−9s,+9s] 구간이 97% coverage 달성 ✓ 대응
쿼리 생성 타당성 모달리티별 캡션 사용은 LongVALE·Watch&Listen과 동일한 방식. 14.3% 노이즈 필터링 ✓ 대응

Reviewer fm3t

지적대응결과
Incremental contribution 쿼리 선호도 70%, 구간 선호도 95% (Figure 5, 7) ✓ 대응
Heuristic filtering 근거 τ = 0.20~0.40 ablation table 제시. 질적 예시 포함 ✓ 대응
댓글 희소한 경우 구독자 100만+ 채널 기반. "w/o comments" 변형도 25% 선호 ✓ 대응
Training 실험 부재 A100 1장으론 불가, H100 8장+ 필요. Compute 한계 인정 ✗ 미해소 → 개선 필요

Reviewer BKiQ

지적대응결과
단일 모달리티 캡션 Mixed category 제안: balance ratio R + absolute floor M 기준으로 분류. Visual+Audio 동시 캡션 생성 방식 제시. Ablation 결과 포함 △ 리버탈에서만 언급 → 실제 구현 필요
중복 댓글 처리 같은 타임스탬프 내 YouTube 좋아요 수 1위 댓글만 유지 ✓ 대응
MLLM bias/hallucination 댓글 grounding이 generic 설명을 구체적 intent로 바꿈. 예시로 증명 ✓ 대응
데이터 규모 Pipeline utility는 human eval로 검증. 코드 공개로 커뮤니티 확장 가능 △ 부분 대응
메타 리뷰어 최종 판단:
리버탈로 일부 우려는 해소됐지만, human evaluation의 논리적 불일치는 리버탈에서 전혀 언급하지 않았다. Section 4.5는 "search intent"를 검증한다고 주장하고, Appendix B.1의 실제 지시문은 "more representative and more informative"를 묻는다. 이 불일치가 거절의 결정적 이유였다.

개선 계획: 다섯 가지 방향

1 Visual Query 추가

예전에 논의했던 대로 visual query를 활용하는 것이 새로운 contribution 중 하나가 될 수 있다. 이미지나 클립 형태로 입력을 줘서 사용자가 원하는 결과를 더 정확하게 찾을 수 있다는 접근이다. 텍스트 쿼리만 존재하는 기존 VMR 태스크에서 벗어나 새로운 태스크를 정의하면, 데이터 규모가 작아도 "새로운 태스크의 첫 데이터셋"으로 포지셔닝이 가능하다.

방법론은 아직 구체화 필요. Visual query를 어떻게 생성하고 어떤 모델로 평가할지 방향을 먼저 잡아야 한다.

2 Mixed Modality 본론 편입

리버탈에서 언급했던 부분인데, 이번에는 실제로 구현해서 본론에 제대로 넣는다. 현재 파이프라인은 댓글을 Visual 또는 Audio 둘 중 하나로만 처리한다. 그러나 "the moment the piano kicks in right as he scores the goal" 같은 댓글은 두 모달리티가 동시에 관여한다.

Mixed category를 Appendix A.3에서 Section 3으로 올리고, Mixed query에 대한 평가를 별도로 포함한다. 현재 이를 잘 처리하는 모델이 없다는 것 자체도 하나의 contribution이 될 수 있다.

3 LanguageBind Downstream Training

현재 논문은 기존 모델을 zero-shot으로 평가하는 데 그친다. 논문 실험에서 가장 강한 성능을 보인 LanguageBind(visual R@10 62.7)를 TCVP로 fine-tuning해서 실제 학습 효과를 보여준다.

왜 LanguageBind인가?
Visual/Audio 멀티모달을 동시에 지원해서 TCVP 데이터셋 특성과 잘 맞는다. 이미 Table 2에서 검증된 가장 강한 베이스라인이므로 narrative의 일관성도 유지된다.

아래 세 조합을 비교한다:

  1. QVHighlights only — 기존 데이터셋 baseline
  2. QVHighlights + TCVP — TCVP 추가 효과
  3. TCVP only — 파이프라인 데이터만의 가능성

4 Human Evaluation 재설계

메타 리뷰어가 가장 크게 지적한 부분이다. 현재 moment selection 지시문은 "more representative and more informative"를 묻고 있어 논문의 핵심 클레임인 search intent 검증과 불일치한다.

"Which moment would you actually want to search for?"를 직접 묻는 방식으로 수정하고, 샘플 수도 40개에서 100개 이상으로 늘린다. Cohen's κ도 함께 보고한다. 이 부분이 해소되면 댓글과 캡션으로 쿼리를 생성하는 타당성 지적도 함께 해소된다.

5 데이터셋 규모 확장

현재 300 videos / 4,500 queries로, 비교 대상인 LongVALE(8,411 videos)에 비해 규모가 크게 작다. 파이프라인이 이미 완성되어 있으므로 추가 수집을 통해 규모를 늘린다. 목표: 300 → 1,000+ videos.
