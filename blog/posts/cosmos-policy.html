Cosmos Policy: 비디오 생성 모델을 로봇 정책으로 파인튜닝하기

아키텍처 변경 없이 대규모 비디오 파운데이션 모델을 로봇 제어 정책으로 변환하는 단일 단계 파인튜닝 접근법과, World Model을 활용한 플래닝까지.

Video · Multimodal · Feb 23, 2026 · 15 min read

배경: 왜 비디오 모델인가?

최근 로봇 조작 정책의 주류는 Vision-Language-Action(VLA) 모델이다. RT-2, OpenVLA, π₀.₅ 등은 대규모 이미지-텍스트 쌍으로 사전학습된 VLM을 백본으로 삼아 로봇 시연 데이터에 파인튜닝한다. 이들은 다양한 의미적 개념에 대한 강한 일반화 능력을 보이지만, 정적인 이미지-텍스트 쌍에서 학습하기 때문에 시간적 인과관계(temporal causality)나 암묵적 물리(implicit physics)를 배우는 데는 한계가 있다.

반면 비디오 생성 모델은 수백만 개의 동영상으로부터 물체의 움직임, 물리적 상호작용, 시간적 흐름을 자연스럽게 학습한다. 이 논문은 이렇게 묻는다: 비디오 모델의 시공간 사전(spatiotemporal priors)을 로봇 제어에 어떻게 활용할 수 있을까? 그 답이 바로 Cosmos Policy다.

1
입력: 여러 카메라 + 언어 명령 ▼
로봇에 달린 여러 대의 카메라 영상(손목, 정면 등)과 "접시 위에 물건 올려"처럼 무엇을 해야 하는지 알려주는 텍스트가 함께 입력된다.
2
출력 ①: 액션 청크 — 지금 당장 뭘 할지 ▼
로봇 팔이 앞으로 몇 초 동안 어떻게 움직여야 하는지(각 관절 각도 시퀀스)를 한 번에 예측한다. 매 프레임마다 계산하지 않고 "덩어리(chunk)"로 예측하므로 움직임이 부드럽다.
3
출력 ②: 미래 상태 — 그 액션 후 어떤 모습일지 ▼
액션을 실행하고 나면 카메라에 무엇이 찍힐지(미래 이미지)를 예측한다. 일종의 "상상"으로, 이걸 잘 예측할수록 더 좋은 계획을 세울 수 있다.
4
출력 ③: 가치 — 이 액션이 성공으로 이어질 확률 ▼
예측한 미래 상태가 얼마나 "좋은" 상태인지를 0~1 사이 숫자로 나타낸다. 여러 액션 후보 중 이 값이 가장 높은 것을 선택하면 성공률이 올라간다.

핵심 아이디어: Latent Frame Injection

기존 비디오 기반 로봇 정책 연구들의 문제는 복잡성이었다. "비디오 모델 파인튜닝 → 별도 액션 모듈 학습"처럼 여러 단계의 훈련을 요구하거나, 역동역학 모델(inverse dynamics model), 별도의 액션 디퓨저 등 새로운 아키텍처 컴포넌트를 추가하는 방식이 대부분이었다.

Cosmos Policy의 핵심 아이디어는 단순하면서 강력하다. 아키텍처를 전혀 수정하지 않고, 로봇의 액션, 고유감각(proprioception), 상태 가치(value)를 모두 잠재 프레임(latent frame)으로 인코딩하여 비디오 모델의 잠재 확산 시퀀스(latent diffusion sequence)에 직접 삽입한다.

핵심: 액션은 "보이지 않는 비디오 프레임"이다. 로봇이 취할 행동을 잠재 공간의 프레임으로 표현하고, 비디오 디퓨전 모델이 그것을 생성하도록 학습시킨다.

구체적으로 Cosmos-Predict2-2B는 Wan2.1 VAE를 사용해 비디오를 (1+T') × H' × W' × 16 크기의 잠재 시퀀스로 압축한다. Latent frame injection은 다음과 같이 이루어진다:

  1. 빈(all-zero) 이미지를 입력 시퀀스에 삽입하여 새 모달리티의 자리를 만든다.
  2. VAE가 이를 잠재 프레임으로 인코딩한다.
  3. 해당 잠재 프레임을 정규화된 로봇 액션/고유감각/가치값으로 덮어쓴다.
  4. 모델은 이 시퀀스 전체를 디노이징하도록 학습된다.
행 1
카메라 영상을 숫자 배열로 압축 ▼
카메라로 찍은 이미지들을 VAE(압축 네트워크)로 줄여서 작은 숫자 격자(잠재 프레임)로 만든다. 또한 액션·고유감각·가치를 넣을 빈 자리(회색 박스)도 함께 준비한다.
행 2
빈 자리에 새 정보를 직접 덮어쓰기 ▼
준비해 둔 빈 자리에 로봇 팔 관절 각도(고유감각), 취할 액션, 미래 상태 가치를 정규화해서 채워 넣는다. 이게 바로 Latent Frame Injection. 별도 모듈 없이 그냥 덮어쓰는 것이 핵심이다.
행 3
노이즈 추가 → 원본 복원 학습 ▼
예측해야 할 프레임들(액션, 미래 이미지, 가치)에 랜덤 노이즈를 섞고, 현재 관측(클린 프레임)을 조건으로 노이즈를 제거하는 훈련을 한다. 이것이 디퓨전 모델의 학습 방식 그대로다.

예를 들어 3대의 카메라(top-down 2대 + 손목 1대)를 가진 ALOHA 로봇의 경우, 잠재 시퀀스는 총 11개의 프레임으로 구성된다: 플레이스홀더, 현재 고유감각, 손목 카메라, 1·2번 카메라, 액션 청크, 미래 고유감각, 미래 손목 카메라, 미래 1·2번 카메라, 미래 상태 가치. 이 순서는 정확히 (s, a, s', V(s'))를 나타낸다.

Joint Training: 정책 + World Model + Value Function

Cosmos Policy는 하나의 모델로 세 가지 기능을 동시에 학습한다.

  • Policy π(a, s', V(s') | s): 현재 상태에서 액션, 미래 상태, 가치를 예측
  • World Model p(s', V(s') | s, a): 현재 상태와 액션으로 미래를 예측
  • Value Function p(V(s') | s, a, s'): 미래 상태의 예상 누적 보상을 예측

각 훈련 배치에서 50%는 시연 데이터(policy 학습), 나머지 50%는 롤아웃 데이터(world model/value function 학습)에 사용된다. 어떤 잠재 프레임이 conditioning되고 어떤 프레임이 denoising 타겟이 되느냐에 따라 세 함수 중 어느 것을 훈련하는지가 결정된다.

50%
Policy 훈련 — "지금 상태만 보고 뭘 할지 예측" ▼
현재 카메라 영상(s)만 주어진 상태에서, 액션·미래 영상·가치를 동시에 예측하도록 훈련한다. 시연 데이터셋의 성공 사례들로 학습한다.
25%
World Model 훈련 — "이렇게 행동하면 어떻게 될지 예측" ▼
현재 상태(s)와 액션(a)이 주어졌을 때, 미래 영상과 가치가 어떻게 될지를 예측한다. 롤아웃 데이터(성공+실패 모두)로 학습해서 실제 결과를 배운다.
25%
Value Function 훈련 — "이 결과가 얼마나 좋은지 판단" ▼
현재 상태(s), 액션(a), 미래 상태(s')가 모두 주어진 상태에서 그 결과가 성공일지 실패일지(0~1)를 예측한다. 이 값을 기준으로 플래닝 시 어떤 액션이 더 좋은지 선택한다.

흥미로운 점은 보조 학습 목표(auxiliary objectives)가 중요하다는 것이다. 정책을 단순히 p(a|s)가 아닌 p(a, s', V(s')|s)로 학습시키면 성능이 유의미하게 향상된다. LIBERO에서 auxiliary loss를 제거하면 평균 성공률이 1.5% 하락, 사전학습 없이 처음부터 학습하면 3.9% 하락한다.

Model-Based Planning

시연 데이터만으로 학습한 world model은 성공 사례에만 노출되어 분포 밖(OOD) 상황에서 부정확한 예측을 낼 수 있다. Cosmos Policy는 이를 해결하기 위해 롤아웃 경험(rollout data)을 수집하고, 이를 기반으로 world model과 value function을 추가 파인튜닝한다.

플래닝은 Best-of-N 샘플링으로 구현된다:

  1. 정책에서 N개의 액션 후보를 샘플링
  2. 각 후보에 대해 world model로 미래 상태 예측 (앙상블 3회)
  3. 각 미래 상태에 대해 value function으로 가치 예측 (앙상블 5회)
  4. 가치가 가장 높은 액션을 실행

여기서 dual deployment를 사용한다. 원래의 Cosmos Policy 체크포인트는 "정책 모델"로, 롤아웃 데이터로 파인튜닝된 체크포인트는 "플래닝 모델"로 분리하여 사용한다. 이를 통해 world model이 정책의 실제 분포에서 수집된 데이터로 학습될 수 있다.

특히 RoboCasa에서의 결과가 인상적이다. Cosmos Policy는 태스크당 50개의 시연 데이터만으로 67.1%의 평균 성공률을 달성, 300개 이상의 데이터를 사용한 경쟁 모델들을 모두 앞섰다. 데이터 효율성 측면에서 뚜렷한 강점을 보인다.

실제 ALOHA 로봇 평가

4개의 bimanual 조작 태스크(plate에 물체 올리기, 셔츠 접기, 사탕 모으기, 지퍼백에 사탕 넣기)에서 Cosmos Policy는 평균 93.6%의 점수로 모든 비교 방법을 능가했다. 특히 높은 action multimodality가 요구되는 "put candies in bowl"과 밀리미터 단위의 정밀도가 필요한 "put candy in ziploc bag"에서 두각을 나타냈다.

VLA 모델인 π₀.₅는 ziploc 슬라이더를 잡는 데 실패하는 경우가 많았고, OpenVLA-OFT+는 L1 regression 기반 액션 예측으로 인해 multimodal한 grasp sequence에서 두 사탕 사이를 어정쩡하게 향하는 실패 패턴을 보였다.

Model-based planning을 적용하면 두 어려운 태스크에서 평균 12.5점 향상을 확인했다. 아래 그림에서 볼 수 있듯이, 롤아웃 데이터로 파인튜닝된 world model은 실패 상황을 미리 예측하여 더 나은 액션을 선택할 수 있게 된다.

위
파인튜닝 전: 실패를 예측 못 함 ▼
시연 데이터(성공 사례만)로만 학습한 World Model은 "이렇게 하면 잘 될 것"이라고 장밋빛 예측만 한다. 슬라이더를 놓치는 실패를 상상조차 못하기 때문에, 나쁜 액션을 걸러내지 못한다.
아래
파인튜닝 후: 실패를 미리 상상함 ▼
실패 롤아웃 데이터를 추가로 학습하면 "이렇게 하면 슬라이더를 놓칠 것 같다"는 예측이 가능해진다. 덕분에 Best-of-N 플래닝에서 그 액션을 걸러내고 더 안전한 액션을 선택할 수 있다.

내 생각

이 논문에서 가장 인상적인 부분은 단순함(simplicity)이다. "액션을 잠재 프레임으로 표현한다"는 아이디어 하나로 아키텍처 수정 없이 정책·world model·value function을 통합한 것은 elegance의 전형이다.

내가 연구하고 있는 query-adaptive 멀티모달 비디오 이해와도 연결점이 있다. Cosmos Policy는 비디오 모델이 시각 정보 외에 액션이라는 새로운 "채널"을 처리하도록 파인튜닝하는데, 이는 결국 어떤 쿼리(태스크 명세)에 따라 관련 정보를 선택적으로 처리하는 문제와 맞닿아 있다. 로봇 정책의 "어떤 행동을 취해야 하는가"는 곧 "어떤 시각 정보가 지금 중요한가"와 분리될 수 없다.

한계로 언급된 플래닝 속도(액션 청크 하나에 약 5초)는 실용적인 배포에서 분명한 병목이다. 특히 동적인 환경에서는 치명적일 수 있다. Consistency model이나 flow matching을 활용한 denoising step 감소, 또는 더 효율적인 search tree 구성이 후속 연구의 핵심 방향이 될 것 같다. 실제로 논문에서 1-step denoising만으로도 RoboCasa에서 66.4%를 달성한 결과는 희망적이다.
