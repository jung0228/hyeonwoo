최대 우도 추정(MLE)과 최대 사후 추정(MAP): 파라미터를 추정하는 두 관점

관측 데이터로부터 모델 파라미터를 찾는 두 가지 방법론 — 빈도주의 관점의 MLE와 베이즈주의 관점의 MAP가 어떻게 연결되는지, 그리고 딥러닝의 손실 함수와 정규화가 이 두 개념에서 어떻게 자연스럽게 도출되는지 정리한다.

Math · Mar 4, 2026 · 18 min read

파라미터 추정이란?

확률 모델을 사용한다는 것은, 데이터가 어떤 파라미터 $\theta$를 가진 분포 $P(X \mid \theta)$에서 생성됐다고 가정하는 것이다. 동전이라면 앞면 확률 $p$, 가우시안 분포라면 평균 $\mu$와 분산 $\sigma^2$, 신경망이라면 수백만 개의 가중치가 $\theta$에 해당한다.

문제는 $\theta$를 모른다는 것이다. 우리가 가진 것은 데이터 $\mathcal{D} = \{x_1, \ldots, x_n\}$뿐이다. 이 데이터로부터 $\theta$를 어떻게 추정할 것인가 — 이것이 파라미터 추정(parameter estimation) 문제다.

크게 두 관점이 있다. 빈도주의(frequentist)는 $\theta$를 고정된 미지의 상수로 보고 데이터만으로 추정한다. 베이즈주의(Bayesian)는 $\theta$도 확률 변수로 보고, 데이터를 관측하기 전의 사전 믿음(prior)을 데이터로 업데이트한다. MLE는 전자, MAP는 후자의 대표적 방법이다.

우도 함수 (Likelihood)

MLE를 이해하려면 먼저 우도(likelihood)의 개념이 중요하다. $P(X \mid \theta)$는 두 가지 관점으로 읽을 수 있다.

  • 확률(probability): $\theta$가 고정됐을 때, $X = x$가 얼마나 자주 나오는가
  • 우도(likelihood): $X = x$가 관측됐을 때, $\theta$라는 파라미터가 얼마나 그럴듯한가

두 관점 모두 같은 함수 $P(X \mid \theta)$를 쓰지만, 무엇을 변수로 보는지가 다르다. 우도로 볼 때는 $x$가 고정이고 $\theta$가 변수다. 이를 명시하기 위해 우도 함수를 다음과 같이 쓴다.

우도 함수 (Likelihood Function)

$$\mathcal{L}(\theta \,;\, \mathcal{D}) = P(\mathcal{D} \mid \theta) = \prod_{i=1}^{n} P(x_i \mid \theta)$$

데이터 포인트가 독립적으로 생성됐다(i.i.d.)고 가정할 때, 전체 우도는 각 포인트의 우도의 곱이다.

여기서 $\prod$로 곱하는 이유가 i.i.d. 가정이다. 각 $x_i$가 독립이면 결합 확률은 주변 확률의 곱이다. 이 곱은 $n$이 커질수록 극도로 작아져 수치적으로 불안정해진다. 그래서 대신 로그 우도(log-likelihood)를 사용한다.

로그 우도 (Log-Likelihood)
$$\ell(\theta) = \log \mathcal{L}(\theta\,;\,\mathcal{D}) = \sum_{i=1}^{n} \log P(x_i \mid \theta)$$

$\log$는 단조 증가 함수이므로 $\mathcal{L}$을 최대화하는 $\theta$와 $\ell$을 최대화하는 $\theta$는 동일하다. 곱이 합으로 바뀌어 수치 안정성도 높아진다.

최대 우도 추정 (MLE)

최대 우도 추정 (Maximum Likelihood Estimation)

$$\hat{\theta}_{MLE} = \arg\max_{\theta} \, \ell(\theta) = \arg\max_{\theta} \sum_{i=1}^{n} \log P(x_i \mid \theta)$$

관측 데이터를 가장 그럴듯하게 만드는 파라미터를 선택한다.

예시 1: 동전 던지기 (Bernoulli)

동전을 $n$번 던져 $k$번 앞면이 나왔다. 앞면 확률 $p$의 MLE를 구해보자.

Bernoulli MLE 풀이
$$\ell(p) = k \log p + (n - k) \log(1 - p)$$ $$\frac{d\ell}{dp} = \frac{k}{p} - \frac{n-k}{1-p} = 0 \quad \Rightarrow \quad \hat{p}_{MLE} = \frac{k}{n}$$

결과는 직관적으로 당연하다. 앞면이 나온 비율이 MLE다. 수학이 "관측한 것을 그대로 믿어라"는 상식을 도출한 것이다.

베르누이 우도 함수 $L(p) = p^k(1-p)^{n-k}$. 두 관측 모두 $n=10$으로 동일하지만 $k$가 다르면 피크 위치가 달라진다.

예시 2: 가우시안 분포

$x_1, \ldots, x_n \overset{\text{iid}}{\sim} \mathcal{N}(\mu, \sigma^2)$일 때 $\mu$와 $\sigma^2$의 MLE를 구하면:

Gaussian MLE 결과
$$\hat{\mu}_{MLE} = \frac{1}{n}\sum_{i=1}^n x_i \qquad (\text{표본 평균})$$ $$\hat{\sigma}^2_{MLE} = \frac{1}{n}\sum_{i=1}^n (x_i - \hat{\mu})^2 \qquad (\text{편향 추정량})$$

평균의 MLE는 표본 평균이다. 분산의 MLE는 $\frac{1}{n}$으로 나누는 편향(biased) 추정량임에 주의하자. 불편 추정량은 $\frac{1}{n-1}$로 나누는데, MLE가 반드시 불편 추정량을 주진 않는다.

MLE의 한계: 데이터가 적을 때 과적합(overfitting)하기 쉽다. 동전을 3번 던져 3번 모두 앞면이 나왔다면 MLE는 $\hat{p} = 1$이다. 하지만 우리는 이 동전이 공정할 가능성도 있다는 것을 안다. 이 사전 지식을 모델에 반영하는 방법이 MAP다.

베이즈 정리와 사후 확률

베이즈 정리는 $\theta$에 대한 믿음을 데이터로 업데이트하는 공식이다.

베이즈 정리 (Bayes' Theorem)

$$P(\theta \mid \mathcal{D}) = \frac{P(\mathcal{D} \mid \theta)\, P(\theta)}{P(\mathcal{D})}$$

$P(\theta)$: 사전 분포(prior) — 데이터를 보기 전 $\theta$에 대한 믿음
$P(\mathcal{D} \mid \theta)$: 우도(likelihood) — 파라미터 $\theta$ 하에서 데이터가 나올 확률
$P(\theta \mid \mathcal{D})$: 사후 분포(posterior) — 데이터를 본 후 $\theta$에 대한 믿음
$P(\mathcal{D})$: 증거(evidence) — 정규화 상수, $\theta$에 무관

$P(\mathcal{D})$는 $\theta$에 무관한 상수이므로, 최적화 목적으로는 무시할 수 있다:

$$P(\theta \mid \mathcal{D}) \propto P(\mathcal{D} \mid \theta)\, P(\theta)$$

사후 확률 ∝ 우도 × 사전 확률

직관적으로, 베이즈 추론은 "과거 경험(prior)과 새 증거(likelihood)를 결합해 믿음을 업데이트한다." 데이터가 많아질수록 우도가 prior를 압도하고 MLE와 수렴한다.

3번 모두 앞면이 나왔을 때(우도 ↗): MLE는 $\hat{p}=1$이지만, 사전 분포가 극단적인 추정을 당겨 MAP는 $\hat{p}=0.8$.

최대 사후 추정 (MAP)

최대 사후 추정 (Maximum A Posteriori Estimation)

$$\hat{\theta}_{MAP} = \arg\max_{\theta}\, P(\theta \mid \mathcal{D}) = \arg\max_{\theta}\left[\sum_{i=1}^n \log P(x_i \mid \theta) + \log P(\theta)\right]$$

사후 확률을 최대화하는 파라미터를 선택한다. MLE에 $\log P(\theta)$ 항이 추가된 형태다.

MLE와 MAP의 차이는 $\log P(\theta)$ 항 하나다. 사전 분포 $P(\theta)$가 균등 분포(uniform)라면 $\log P(\theta)$는 상수이고, MAP와 MLE는 동일한 해를 준다. 즉 MLE = uniform prior인 MAP다.

사후 분포(빨강) = 우도(파랑) + 사전 분포(회색)의 합산. MAP는 MLE와 사전 피크(θ=0) 사이에서 균형을 잡는다.

MAP는 데이터가 말하는 것(우도)과 우리가 이미 아는 것(사전 분포)을 균형 있게 반영한 추정이다.

정규화의 기원: 사전 분포의 선택

딥러닝에서 흔히 쓰이는 L2 정규화(weight decay)와 L1 정규화(Lasso)는 사실 MAP 추정의 특수한 경우다. 사전 분포를 어떻게 선택하느냐에 따라 정규화 형태가 결정된다.

가우시안 사전 분포 → L2 정규화

각 파라미터가 독립적으로 $\theta_j \sim \mathcal{N}(0, \tau^2)$라고 가정하면:

Gaussian Prior → L2 Regularization
$$\log P(\theta) = -\frac{1}{2\tau^2}\sum_j \theta_j^2 + \text{const} = -\frac{1}{2\tau^2}\|\theta\|_2^2 + \text{const}$$ $$\hat{\theta}_{MAP} = \arg\max_{\theta}\left[\sum_i \log P(x_i \mid \theta) - \frac{1}{2\tau^2}\|\theta\|_2^2\right]$$ $$= \arg\min_{\theta}\left[\underbrace{-\sum_i \log P(x_i \mid \theta)}_{\text{NLL}} + \underbrace{\lambda\|\theta\|_2^2}_{L2 \text{ 정규화}}\right], \quad \lambda = \frac{1}{2\tau^2}$$

딥러닝에서의 weight decay가 정확히 이것이다. 사전 분포의 분산 $\tau^2$이 클수록(파라미터에 대한 사전 확신이 약할수록) $\lambda$가 작아지고 정규화 강도가 약해진다.

라플라스 사전 분포 → L1 정규화

각 파라미터가 $\theta_j \sim \text{Laplace}(0, b)$를 따른다고 가정하면:

Laplace Prior → L1 Regularization
$$\log P(\theta) = -\frac{1}{b}\sum_j |\theta_j| + \text{const} = -\frac{1}{b}\|\theta\|_1 + \text{const}$$ $$\hat{\theta}_{MAP} = \arg\min_{\theta}\left[-\sum_i \log P(x_i \mid \theta) + \lambda\|\theta\|_1\right], \quad \lambda = \frac{1}{b}$$

라플라스 분포는 0 근방에서 뾰족한 피크를 가지고 꼬리가 두꺼운 분포다. 이는 "파라미터 대부분이 0에 가깝고, 일부는 클 수도 있다"는 사전 믿음이다. L1 정규화가 희소(sparse) 해를 유도하는 이유가 바로 이 라플라스 사전 분포의 성질에서 비롯된다.

두 사전 분포의 형태 비교. 라플라스는 0 근방이 급격해 파라미터를 정확히 0으로 만들지만(희소성), 가우시안은 부드럽게 0 방향으로 수축시킨다.

사전 분포 $\log P(\theta)$ 정규화 항 효과
균등 (Uniform) 상수 없음 MLE와 동일
가우시안 $\mathcal{N}(0, \tau^2)$ $-\frac{1}{2\tau^2}\|\theta\|_2^2$ L2 (Weight Decay) 파라미터를 0으로 당김 (수축)
라플라스 $\text{Laplace}(0, b)$ $-\frac{1}{b}\|\theta\|_1$ L1 (Lasso) 파라미터를 0으로 만들어 희소성 유도

딥러닝 손실 함수의 MLE 해석

딥러닝에서 쓰이는 대부분의 손실 함수는 MLE로부터 자연스럽게 도출된다. "어떤 분포를 가정하는가"가 손실 함수를 결정한다.

교차 엔트로피 손실 = Categorical NLL

분류 문제에서 레이블 $y_i \in \{1, \ldots, C\}$가 $P(y_i = c \mid x_i;\, \theta) = \text{softmax}(f_\theta(x_i))_c$를 따른다고 가정하면:

Cross-Entropy = Categorical Negative Log-Likelihood
$$-\log P(y_i \mid x_i; \theta) = -\sum_{c=1}^C \mathbf{1}[y_i = c]\, \log \hat{p}_c(x_i;\theta)$$ $$\text{NLL} = -\sum_{i=1}^n \log P(y_i \mid x_i;\theta) = \sum_{i=1}^n H(y_i^{\text{one-hot}},\, \hat{p}(x_i;\theta))$$

분류에서 cross-entropy loss를 최소화하는 것은 categorical likelihood를 최대화하는 것과 완전히 동일하다. 손실 함수 선택에 이미 확률 모델에 대한 가정이 내재되어 있는 것이다.

평균 제곱 오차 (MSE) = Gaussian NLL

회귀 문제에서 $y_i \sim \mathcal{N}(f_\theta(x_i), \sigma^2)$라고 가정하면:

MSE = Gaussian Negative Log-Likelihood
$$-\log P(y_i \mid x_i;\theta) = \frac{(y_i - f_\theta(x_i))^2}{2\sigma^2} + \frac{1}{2}\log(2\pi\sigma^2)$$ $$\text{NLL} \propto \sum_{i=1}^n (y_i - f_\theta(x_i))^2 = \text{MSE}$$

$\sigma^2$이 상수라면 NLL 최소화는 MSE 최소화와 동치다. MSE 손실을 쓰는 것은 암묵적으로 잔차가 가우시안 분포를 따른다고 가정하는 것이다. 이상치(outlier)에 MSE가 취약한 이유도 여기 있다 — 가우시안 분포는 꼬리가 얇아 큰 오차에 과도한 패널티를 준다. Huber 손실이나 MAE(L1 손실)는 라플라스 분포 가정에 해당하며, 이상치에 더 강건하다.

요약: 손실 함수를 선택할 때 "어떤 노이즈 모델을 가정하는가?"를 생각하면 선택의 근거가 명확해진다. 가우시안 노이즈 → MSE, Bernoulli/Categorical → Cross-entropy, 라플라스 노이즈 → MAE.

마치며

MLE와 MAP는 단순히 "파라미터 추정 방법" 이상의 의미를 가진다. 이 두 개념은 딥러닝 전반에 걸쳐 있는 아이디어들의 출발점이다.

손실 함수는 우도 모델의 선택이고, 정규화는 사전 분포의 선택이다. 배치 정규화, 드롭아웃, 앙상블 등 다양한 기법들도 베이즈 관점에서 해석될 수 있다. 이 틀을 이해하면 새로운 방법론을 접했을 때 "어떤 분포를 가정하는가?", "어떤 사전 지식을 인코딩하는가?"라는 질문을 자연스럽게 던질 수 있다.

다음 포스트에서는 MLE와 MAP의 목적 함수에 등장하는 로그 확률의 합산이 정보이론과 어떻게 연결되는지 — 엔트로피, 교차 엔트로피, KL divergence — 를 다룬다.
